강의 준비하기

강사는 클래스를 진행하기 전에 다음의 준비과정을 진행할 수 있다:

1. 수강생 인원이 모두 문제 없이 서비스에 정상적으로 접속하여 출석했는지 확인 2. 수강생의 이름이 정확한지, 중복된 수강생이 존재하는 확인 > [팁] 중복된 수강생이 있으면 강의 진행중 전체 인원이 각 랩을 정상 수행했는지 체크하기가 어려워지기 때문에 강의 초기에 확인해 주는 것이 좋습니다. 3. 수강생이 원래 등록된 인원보다 더 많은 지 확인 --- - 클래스룸 입장
- 1. 해당 랩의 이름이 표시된다. - 2. 랩툴바 > 클래스룸에서 이용가능한 기능들이 표시된다. - 랩실 > 랩실로 입장된다. - 미션 > 해당 랩의 미션(Instruction)이 표시된다.
- 이론/자료 > 이론/자료 내용이 표시진다. - 1.버튼 클릭시 수강생들에게 상금을 지급할 수 있고, 수강생들의 응답을 한번에 확인, 삭제가 가능하다.
- 설문 > 간단한 퀴즈를 내고 상금을 지급하거나, 응답을 한번에 확인, 삭제를 할 수 있다.
- 응답 > 수강생들이 질문에 대한 응답을 할 수 있다.
- <b>랩 목록</b> > 라운지로 돌아간다.
- <b>그룹 추가</b> > 7. 항목과 같이 새로운 '생성된 조 1'가 추가된다.
- <b>SCORE</b> > 모든 유저의 점수(상금)을 확인할 수 있다.
- <b>일괄 실행</b> > ide 랩인 경우 모두 일괄 실행 시킬 수 있다.
- <b>시작</b> > 타이머가 시작된다.

  • 3. 조의 이름과 수강생들의 랩 진행정도가 보여진다.(수료한 사람수/출석한 사람수/접속한 사람수/전체사람수)

    • 수료 > 랩의 모든 체크포인트를 완수함.
    • 출석 > 해당 랩실에 접속하여 3분 이상 작업을 수행함.
    • 접속 > 해당 랩실에 접속함.
    • 전체 > 전체 사람 수를 나타냄.
  • 4. 수강생이 응답 시 프로필 우측 상단에 표시한다.(ex 3)
  • 5. 수강생의 이미지에 마우스를 올리면 수강생의 정보 수정이 가능한 아이콘이 나타난다.
    <수강생 정보수정 화면>
  • 6. 해당 수강생의 체크포인트 완수정도,출석,접속 여부를 확인 할 수 있다.

  • 7. 그룹 추가 시 새로 추가된 조

    • 수강생 이미지를 드레그앤드롭으로 조를 이동시킬수 있다.
강의 진행하기

  • 도움요청

    • 도움요청을 클릭하게 되면 해당 수강생의 이미지 상단에 '헬프미' 뱃지가 표시된다.

  • 수강생 이미지를 클릭하면 사진과 같이 판넬이 열린다.

    • 1. 선택한 수강생의 이름과 이메일을 확인할 수 있다.
    • 2. 선택한 수강생의 Hash 정보를 확인할 수 있다.
    • 3. 선택한 수강생의 체크포인트 완수정도를 파악할 수 있고 체크포인트 관리도 가능하다.

    • 4. 버튼 기능

      • 랩실보기 > 선택한 수강생의 랩실을 확인할 수있다.
      • 랩실 초기화 > 선택한 수강생의 랩실을 초기화 시킨다.
      • 상금 지급 > 선택한 수강생에게만 상금을 지급/차감할 수 있다.

    • 5. 랩실에서 결과제출을 하게되면, 랩실의 결과(터미널)이 보여진다.

      • 결과제출

  • 상금

    • 상금을 기준으로 순위가 트로피로 수여된다.

  • 도구별 기타 기능

    • 이벤트스토밍

      • Code Preview

        • 생성된 코드를 확인할 수 있다.

      • Download Archive

        • Template를 정하고 다운로드 받을 수 있다.

      • Project IDE

        • 모델링한 코드가 새창(ide)으로 열린다.

      • SAVE

        • save 버튼을 클릭하면 json 파일로 저장된다.
강의 평가 및 결과 정리
  • 실습 진행 후 수강생들의 진행정도를 확인할 수 있다.

  • (수료한 사람수/출석한 사람수/접속한 사람수/전체사람수)

    • 수료 > 랩의 모든 체크포인트를 완수함.
    • 출석 > 해당 랩실에 접속하여 3분 이상 작업을 수행함.
    • 접속 > 해당 랩실에 접속함.
    • 전체 > 전체 사람 수를 나타냄.
교육생 편의 기능

  • 좌측 판넬 크기 조절

    • 좌측 상단의 버튼을 클릭하면 판넬의 크기를 조절해서 사용할 수 있다.(크게, 작게 또는 가리기)

  • 새터미널

    • 새터미널을 클릭하고 터미널의 이름을 입력하면 해당 이름으로 새터미널이 생성된다.
    • 이름 입력

  • 포트 열기

    • labs > 포트열기 > 포트넘버를 선택하면 지정된 포트로 열기가 가능하다.

  • 포트 확인 - 현재 사용중인 포트를 확인하여, 포트충돌을 예방할 수 있다.

강의 준비하기

  • 교육생 전체 출석확인 및 이름수정 : 수강생 인원이 모두 문제 없이 서비스에 정상적으로 접속하여 출석했는지 확인합니다. 수강생의 이름이 정확한지, 중복된 수강생이 존재하는지 확인합니다.

    [팁] 중복된 수강생이 있으면 강의 진행중 전체 인원이 각 랩을 정상 수행했는지 체크하기가 어려워지기 때문에 강의 초기에 확인해 주는 것이 좋습니다.

  • 그룹(조)편성 : 그룹을 추가하고, 교육생들을 할당 시킬 수 있습니다.
  • 랩 미션 및 이론/자료 : 해당 랩의 미션과 이론/자료를 확인할 수 있습니다.
  • 설문조사 : 간단한 설문조사를 하고 응답을 받아볼 수 있습니다.

코스 > 강의 입장 > 라운지 > 클래스룸으로 입장

- 클래스룸으로 입장했을 때 보여지는 화면입니다.

1. 교육생 전체 출석확인 및 이름수정

  • 그룹명(미배정)과 해당 랩의 참석도(수료한사람수/출석한사람수/접속한사람수/전체사람수)를 확인 할 수 있습니다.

    수료한사람수 : 해당 랩의 체크포인트를 모두 완수한 사람
    출석한사람수 : 해당 랩에서 3분이상 작업을 진행한 사람
    졉속한사람수 : 해당 랩에 접속한 사람
    전체사람수 : 전체 사람수

  • 교육생의 이름을 수정할 수 있습니다.

    • 수정할 교육생의 이미지에 마우스를 올리고 생성된 연필아이콘을 누르게 되면 교육생 정보 수정화면이 열리게 됩니다.

      [교육생정보수정 화면]

2. 그룹(조)편성

  • 랩툴바 > 그룹 추가 버튼을 클릭하면, 사진과 같이 새로운 '생성된 조 1'가 추가되며, 그룹명을 수정할 때는 그룹명(ex 생성된 조 1)을 더블 클릭후 수정한다. 교육생 이미지를 드레그앤드롭으로 조를 이동시킬수 있습니다.

3. 랩 미션 설명 및 이론/자료 설명

  • 랩툴바 > 미션 > 해당 랩에서 수행할 미션이 표시됩니다.

  • 랩툴바 > 이론/자료 > 이론/자료 내용이 표시됩니다.

    • 버튼 클릭시 교육생들에게 상금을 지급할 수 있고, 교육생들의 응답을 한번에 확인, 삭제가 가능합니다.

4. 강의전 수강생의 사전 지식에 대한 설문 조사

  • 랩툴바 > 설문 > 간단한 퀴즈를 내고 상금을 지급하거나, 응답을 한번에 확인, 삭제를 할 수 있습니다.

강의 컨텐츠 제작하기

1. 폴더 구조

  1. 기본적인 폴더 구조는 아래와 같다.

  • 코스이름 폴더

    • classes 폴더

      • running

        • 클래스이름 폴더

          • Class_Metadata.json

    • labs 폴더

      • lab1 폴더

        • instruction.md
        • Lab_Metadata.json
      • lab2 폴더
    • Course_Metadata.json
    • default.png
  1. 최상위 폴더 2.1 Course_Metadata.json

  • json 파일로서 코스에 해당하는 이름과 설명이 들어가 있다.

    {
  "instructorName" : "유엔진",
  "instructorId" : "admin@uengine.org",
  "courseName" : "msaez 에 클래스 생성 방법",
  "courseDesc" : "msaez 에 클래스 생성 방법"
}

    2.2 default.png

  • msaez.io 에서 course 탭을 눌렀을때 나오는 리스트의 이미지 파일이다.
  1. labs 폴더 3.1 labs 폴더는 하위에 lab들을 포함한다.
  • lab 은 클래스 입장시 아래 이미지 처럼 개별 랩으로 만들어진다. lab 폴더명이 파란색 동그라미 안에 그려진 이름으로 표시된다.
  • 예) 첫번째 랩은 labs - default-env 라는 폴더로 생성됨.

  • 랩이름이나, 랩에 대한 상세 정보는 Lab_Metadata.json 에 작성한다.

    3.2 Lab_Metadata.json

    {
    "labName": "[구현] 데이터프로젝션-CQRS",
    // ide, event-storming 를 가진다.
    "tool" : "ide", 
    "labTime" : 10,	
    "labScenario" : "이벤트를 수신하여 READ 모델을 만드시오",
    // event-storming 랩 시작시 실행할 json 스크립트 
    "startArtifact": "start.json",  
    // ide 랩 시작시 템플릿으로 필요한 파일이 있는경우 포함
    "templateFile": "xxx.zip",	
    // 동영상이 있을 경우 포함 
    "video": "https://youtu.be/ipXtftkO5eo",
    // 체크포인트를 넣고 싶을경우 포함한다.
    "checkPoints": [
        {
          "id": "id1",
            "text": "체크포인트 질문하세요",
            "testShell": "test OrderCancelled"
        }
    ],
    // 힌트를 넣고 싶을경우 포함한다.
    "hints": [{
        "text": "",
        "regExp": ""
    }]
}
    이름 설명 사용법
    labName 랩실의 이름
    labScenario 랩실의 간략설명
    tool 랩실 오른쪽에 나오는 툴의 종류 ide, event-storming , quiz
    labTime 랩진행 시간 표시 10 으로 적으면 10분이라고 표시됨
    startArtifact event-storming 랩 시작시 실행할 json 스크립트

tool 별로 체크포인트를 설정하는 방법이 약간씩 다르기 때문에 툴별로 나누어서 설명을 진행한다.

tool : ide

1. regExp

- 가장 기본적인 사용법은 regExp 를 사용하여 결과 제출시 작업하고 있는 터미널을 그대로 가지고와서 체크하는 방법이다. - 아래와 같이 설정을 한 후, 터미널을 연 후, http www.example.com 을 조회한다. - http 명령의 결과로 HTTP/1.1 200 OK 값이 나왔을 텐데, 해당 문구를 바로 비교하여 체크를 한다. ``` "checkPoints": [ { "id": "id1", "text": "regExp 를 사용하여 데이터를 조회한 결과를 체크하시오", "regExp": "HTTP/1.1 200" } ] ```

2. testShell

- 이 설정은 같은 랩실에 test.sh 파일이 존재해야 작동한다. - test.sh 에 쉘 스크립트로 로직을 설정 한 후, 결과값을 비교하여 작동한다. - "testShell": "test [키값]" 방식으로 Lab_Metadata.json 에 설정을 하게 된다. - 여기서 [키값]은 test.sh 에 설정되어있는 키 값이다. ``` "checkPoints": [ { "id": "id2", "text": "testShell 를 사용하여 데이터를 조회한 결과를 체크하시오", "testShell": "test check1" } ] ``` - 이렇게 설정 후 같은 랩실에 test.sh 파일을 만든 후 아래와 같이 설정한다. - test.sh 파일은 ####### (#7개) 를 체크한 후에 test check1 이라는 문구를 인식 한 후, 결과값이 HTTP/1.1 200 이 있는지를 체크하는 로직이다. - 다음 라인에 나오는 부분이 실행하는 명령어이다. - echo "####### finish" 를 test.sh 파일의 가장 하단에 위치하여 테스트가 끝났다는것을 확인시켜준다. ``` echo "####### test check1 : HTTP/1.1 200" http www.example.com echo "####### finish" ``` - 만약 체크를 할것이 두개라면 ,(콤마) 로 구분하면 두가지 조건이 모두 충족되었을때 체크박스에 체크가 된다 ``` echo "####### test OrderCancelled : \"orderStatus\": \"OrderCancelled\" " http localhost:8090/orderStatuses echo "####### test DeliveryStarted : \"orderStatus\": \"OrderCancelled\",\"deliveryStatus\": \"DeliveryStarted\" " http localhost:8090/orderStatuses echo "####### finish" ```

tool : event-storming

event-storming 으로 사용을 할때는 화면상의 내용을 javascript 로 체크를 한다. 체크를 할적에는 inline 으로 스크립트 코드를 바로 작성 할 수도 있고, js 파일을 만들어서 해당 파일을 바로 실행 시키는 방법이 있다. 
"checkPoints": [
        {
            "id": "id3",
            "text": "inline javascript code 를 체크한다.",
            "javascript": "oneOf(t.query('$.Event[*].name'), ['OrderPlaced', 'OrderCanceled'])"
        },
        {
            "id": "id4",
            "text": "파일로 체크포인트를 만드는 방법을 확인한다.",
            "javascript": "isPostPoned.js"
        }
    ],

1. inline javascript code

- 위의 checkPoints 'id3' 을 보면 inline code 가 들어가 있다. - 이벤트를 2개 만들고 이벤트 이름에 'OrderPlaced' 와 'OrderCanceled' 를 넣은 후에 왼쪽 상단에 제출 버튼을 클릭한다. - t.query('$.Event[*].name') 부분에 의해서 화면상의 이벤트를 모두 가져온 후에, 값을 비교하여 정상적으로 들어가 있을 경우 체크박스를 체크하게 된다.

2. make javascript flie

- 위의 checkPoints 'id4' 를 보면 js 파일명이 들어가 있다. 이때 isPostPoned.js 파일은 랩실안쪽에 들어가 있어야한다. - 가능하면 파일명은 해당 결과를 직관할수 있는 명칭으로 만들어 주면 재사용성면에서 편리하다. - js 파일의 내용은 아래와 같다. - result.elements 의 값은 화면상의 이벤트 스토밍한 결과값이고, 그중에서 이벤트를 찾아서 p.p 형식으로 되어있는지 체크하는 로직이다. - 마지막에 returnValue 부분에 최종 결과를 true, false 형식으로 보여주면 된다. - 이벤트를 2개 만들고 이벤트 이름에 'OrderPlaced' 와 '주문됨' 을 넣은 후에 왼쪽 상단에 제출 버튼을 클릭 하여 체크포인트가 작동하는지 확인한다. 
var count = 0;
var postponed = 0;

Object.keys(result.elements).forEach(key => {
    var item = result.elements[key];
    if (item && item._type == 'org.uengine.modeling.model.Event') {
        count++;
        if (item.name.endsWith('d') || item.name.endsWith('됨') || item.name.endsWith('함')) postponed++;
    }
});

returnValue(count == postponed);

  • 위 사진처럼 포트넘버를 8081로 선택하였을 때 8081로 열리는 것을 확인 할수 있습니다.

  • IDE 상단메뉴 > Labs > 포트확인을 클릭하여, 현재 사용중인 포트를 확인할 수 있습니다.

    포트확인을 하기 위해선, 새 터미널을 열고 진행해야합니다.

  • 포트 확인 클릭 시

netstat -anlp | grep :808 명령어를 실행시킵니다 .
netstat 명령어로 실행중인 포트를 조회합니다.
grep :808 명령어로 808 로 시작하는 포트를 조회합니다.

Class 등록

VS Code로 YHtest 폴더를 열은 다음 Class_Metadata.json 파일을 클릭합니다. 
    "teacherId": "",
    "className": "Test Course",
    "description": "Test를 위한 과정",
	"classStartDate": "2020.08.25",
    "classEndDate": "2020.08.25",
    "openClass": true,
    "groupedUsers":[
    ],
    "labsList": [
        "1"
    ]
}

teacherId 는 강사 ID 이므로 강사의 gmail 주소를 적고,

className 에는 개설하고자 하는 강좌의 이름, description 에는 강좌의 대한 설명,

classStartDate 에는 강좌의 시작 날짜, classEndDate 에는 강좌의 종료 날짜,

"labsList"에는 랩의 개수 만큼 목록을 작성합니다. 현재는 랩이 한개 밖에 없기 때문에 "1"만 표기합니다.

"1"에 대한 실습 파일은 labs/1 에 작성됩니다.

Lab 등록

labs/quiz에 들어가서 Lab_Metadata.json 파일을 클릭합니다.

{
    "labName": "Hello World",
    "tool" : "ide",
    "labTime" : 10,
    "labScenario" : "print Hello World",
    "templateFile": "HelloWorld.zip",
    "checkPoints": [
        {
            "text": "Hello World를 출력하세요",
            "testShell": "test"
        }
    ],
    "hints": [
		{
        "text": "Hello World"
        }
	]
}

labName은 실습하려는 실습명을 적고 tool은 개발환경이 필요하기 때문에 ide 라고 적습니다.

labTime은 실습 시간, labScenario는 시나리오에 대해 적어줍니다.

templateFile은 위에서 작성했던 템플릿을 불러오기 위해 "templateFile": "HelloWorld.zip" 을 적어줍니다.

checkPoints 에는 앞서 작성했던 test.sh 파일을 불러오기 위해 "testShell": "test"을 적어줍니다.

강의 정리하기

강사는 강의가 끝난 뒤 다음과 같이 평가 및 결과정리를 할 수 있습니다:

- 강사는 클래스룸에서 교육생들의 해당 랩 진행도를 한 눈에 파악할 수 있습니다.

  • 진행정도(수료한사람수/출석한사람수/접속한사람수/전체사람수) 확인 방법.

    수료한사람수 : 해당 랩의 체크포인트를 모두 완수한 사람
    출석한사람수 : 해당 랩에서 3분이상 작업을 진행한 사람
    졉속한사람수 : 해당 랩에 접속한 사람
    전체사람수 : 전체 사람수
